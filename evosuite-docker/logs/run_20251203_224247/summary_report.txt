Notes for Research
=============================================

EvoSuite Architectural Limitation:
  EvoSuite uses a master-client RMI architecture where the master
  process cannot deserialize Spring framework types (HttpEntity,
  ResponseEntity, etc.) because they are not in its classpath.
  This causes test generation to fail for most Spring controllers.

Expected Behavior:
  - Controllers will likely show 0 generated tests
  - Errors like 'NoClassDefFoundError: org/springframework/http/HttpEntity'
    are expected and documented in the log files
